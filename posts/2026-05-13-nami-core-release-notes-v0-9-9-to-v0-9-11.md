# Nami Core Release Notes: v0.9.9, v0.9.10, v0.9.11

## [0.9.11] - 2026-05-13

### เพิ่มเติม

-   **การเชื่อมต่อ LINE Bot**: เพิ่มโหมดการทำงานใหม่สำหรับการเชื่อมต่อกับแพลตฟอร์ม LINE
    -   เพิ่มคำสั่ง `line` ใน CLI
    -   สร้าง webhook server โดยใช้ Axum พร้อมการตรวจสอบลายเซ็น HMAC-SHA256
    -   รวมเข้ากับ LINE Messaging API สำหรับการตอบกลับข้อความผู้ใช้
-   **การปรับโครงสร้างการเริ่มต้นโปรเจกต์**: ปรับปรุงคำสั่ง `init` ให้เป็นเวิร์กโฟลว์ 4 ขั้นตอน (LLM, Search, Bot, Observability) พร้อมส่วนหัวที่ปรับปรุงให้ดีขึ้นและการจัดกลุ่ม prompt ที่เข้าใจง่าย
-   **การตั้งค่า Observability**: เพิ่มการรองรับการกำหนดค่า URL ของ OpenTelemetry collector (`OTEL_COLLECTOR`) ระหว่างการเริ่มต้น
-   **เอกสารประกอบที่ปรับปรุง**: อัปเดต `README.md` ของโมดูล `modes` ด้วยคำอธิบายที่ครอบคลุมสำหรับโหมดการทำงานปัจจุบันทั้งหมด รวมถึงการรวม LINE ใหม่และคำสั่ง `run` โดยตรง

## [0.9.10] - 2026-05-12

### เพิ่มเติม

-   **การรองรับ Vertex AI**: เพิ่มการรองรับ Vertex AI ของ Google Cloud โดยใช้ Application Default Credentials (ADC) ช่วยให้สามารถรวมเข้ากับโมเดล Gemini ระดับองค์กรได้อย่างราบรื่น
-   **การตั้งค่าภูมิภาค**: เพิ่มฟิลด์ `project_id` และ `location` ใน `ModelConfig` และ `ReflectionConfig` เพื่อรองรับการปรับใช้ LLM ในภูมิภาคและความต้องการด้าน residency ของ Google Cloud
-   **การตั้งค่า Vertex แบบโต้ตอบ**: อัปเดตคำสั่ง `init` เพื่อกำหนดค่าการตั้งค่า Vertex AI แบบโต้ตอบ โดยจะตรวจจับและแจ้งเตือนพารามิเตอร์ Google Cloud ที่จำเป็นโดยอัตโนมัติ

### ปรับปรุง

-   **ตรรกะการโหลดโมเดล**: ปรับโครงสร้าง `load_model` ใหม่เพื่อข้ามการตรวจสอบตัวแปรสภาพแวดล้อม API key ที่บังคับเมื่อใช้ provider "vertex" โดยจะใช้ ADC ระดับระบบแทน
-   **การสืบทอดการตั้งค่า**: ปรับปรุง `load_model_with_fallback` เพื่อให้แน่ใจว่าการตั้งค่าภูมิภาค (`project_id`, `location`) เผยแพร่จากค่าเริ่มต้นทั่วโลกไปยังเอเจนต์พิเศษได้อย่างถูกต้อง
-   **เอกสารประกอบโค้ด**: เพิ่ม `rustdoc` comments ที่ครอบคลุมในโมดูลเอเจนต์หลัก (`src/agent/agent.rs`) เพื่อปรับปรุงความชัดเจนของสถาปัตยกรรมและการบำรุงรักษา

### แก้ไข

-   **ความเข้ากันได้ของ Tool Schema ของ Vertex AI**: แก้ไขข้อกำหนด schema สำหรับเครื่องมือ (`summarize_wiki`, `get_system_status`, `list_todos`, `list_wiki_pages`, `get_wiki_graph`, `sanitize_wiki_vault`, `check_broken_links`) ให้ใช้ struct ที่ว่างเปล่า เพื่อให้มั่นใจถึงความเข้ากันได้กับ schema พารามิเตอร์ `OBJECT` ของ Vertex AI
-   **การแก้ไขไวยากรณ์และคำผิด**: แก้ไขข้อผิดพลาดทางไวยากรณ์ใน `src/tools/todo/mod.rs` (ฟังก์ชัน `todo_tools`) และกู้คืนข้อกำหนด struct `ListTodosArgs` ที่หายไป
-   **การเปลี่ยนแปลงที่ไม่ได้ตั้งใจถูกยกเลิก**: ยกเลิกการเปลี่ยนแปลงโดยบังเอิญใน `src/agent/agent.rs` ซึ่ง `overlap_size` ถูกตั้งค่าเป็น `1` แทนที่จะเป็น `0` อย่างไม่ถูกต้อง

## [0.9.9] - 2026-05-11

### เพิ่มเติม

-   **ตัวบ่งชี้สถานะเซิร์ฟเวอร์**: รวมส่วนประกอบการเชื่อมต่อเซิร์ฟเวอร์แบบเรียลไทม์ใหม่ในแถบด้านข้างของ WebUI เพื่อให้ข้อเสนอแนะด้านสุขภาพของแบ็กเอนด์แบบภาพ
-   **การนำทางประวัติ Prompt**: เปิดใช้งานการนำทางด้วยปุ่มลูกศร (ขึ้น/ลง) ในช่องป้อนข้อมูล WebUI เพื่อเลื่อนดูประวัติ prompt ก่อนหน้า
-   **การเรนเดอร์ Tool-Call**: ใช้ส่วนประกอบ UI เฉพาะ (`ToolAccordion`) ใน WebUI เพื่อแสดงการเรียกใช้เครื่องมือที่มีโครงสร้างและผลลัพธ์ภายในเธรดการแชท

### ปรับปรุง

-   **การนำเสนอผลลัพธ์ของเครื่องมือ**: ปรับปรุงคำแนะนำ persona ของเอเจนต์เพื่อบังคับใช้สรุปผลลัพธ์ของเครื่องมือด้วยภาษาธรรมชาติที่กระชับ ห้ามแสดง JSON ดิบในการตอบกลับการแชท
-   **การเรนเดอร์ Markdown**: ปรับปรุงการเรนเดอร์ `ReactMarkdown` ใน WebUI ด้วยการปรับระยะห่างย่อหน้าให้กระชับยิ่งขึ้น (`prose-p:m-0`) เพื่อให้เลย์เอาต์ข้อความดูสะอาดตาขึ้น
-   **ตรรกะการสตรีม**: ปรับปรุงตัวประมวลผลล่วงหน้าการสตรีมข้อความใน `useChat.ts` เพื่อจัดการการเปลี่ยนผ่านอักขระภาษาไทย/ไม่ใช่ภาษาไทยและรูปแบบตัวทำเครื่องหมายรายการ ป้องกันการรวมคำ
-   **ความคลุมเครือของ Task**: เปลี่ยนชื่อศัพท์เครื่องมือ Todo (จาก "task" เป็น "todo item") เพื่อป้องกันความสับสนกับ task ของ State Manager

### แก้ไข

-   **Module Resolution**: แก้ไขข้อผิดพลาดไวยากรณ์การนำเข้าโมดูลใน `ThreadView.tsx` ที่เกิดจากการจัดการพาธที่ไม่ถูกต้องระหว่างการรวม
